# 需求文档

## 1. 应用概述

- **应用名称**：令狐赵桓的个人主页
- **应用描述**：一个简洁、清新且具备科技感的个人主页原型，采用单页布局，集成个人信息展示与数字分身聊天区，支持访客与数字分身进行实时交互，完全适配移动端设备。

---

## 2. 用户与使用场景

- **目标用户**：访问该主页的任意访客（同学、导师、招聘方、陌生人等）
- **核心使用场景**：
  - 访客浏览个人基本信息，快速了解页面主人的身份与背景
  - 访客通过数字分身聊天区与「令狐赵桓」进行互动问答

---

## 3. 页面结构与核心功能

### 3.1 页面总览（单页布局）

```
个人主页（单页）
├── 页头区域
│   ├── 固定导航栏（可选）
│   ├── 个人头像
│   ├── 姓名展示
│   └── 一句话介绍
├── 主体信息展示区
│   └── 关于我板块
│       ├── 当前身份
│       ├── 当前主要任务
│       ├── 兴趣
│       └── 其他（可扩展备注栏）
└── 数字分身聊天区
    ├── 对话历史显示区
    └── 用户输入框
```

### 3.2 页头区域

**固定导航栏（可选）**
- 页面滚动时导航栏固定于顶部
- 包含锚点链接，可快速跳转至「关于我」和「聊天区」两个模块

**个人头像**
- 展示圆形头像占位图（默认使用姓名首字母或占位图案）
- 头像尺寸在移动端不小于 80px

**姓名展示**
- 显眼展示姓名「令狐赵桓」，字号突出，作为页面视觉焦点

**一句话介绍**
- 固定文案：「一个专注因果推理的算法探索者」
- 位于姓名正下方，字号次于姓名

### 3.3 主体信息展示区 —— 关于我板块

以卡片或分组形式展示以下结构化信息：

| 字段 | 内容 |
|------|------|
| 当前身份 | 计算机专业的研0学生 |
| 当前主要任务 | 毕设和 vibe coding |
| 兴趣 | 运动和游戏 |
| 其他 | 可扩展备注栏，默认展示「姓名由四个字构成」等记忆点，支持后续追加条目 |

- 每个字段以「标签 + 内容」形式呈现，信息层级清晰
- 「其他」栏以列表形式展示，便于后续扩展

### 3.4 数字分身聊天区

**模块定位**
- 独立聊天窗口模块，视觉上与信息展示区明确区分
- 模块标题可标注「和我聊聊」或「数字分身」等提示文案

**对话历史显示区**
- 展示用户与数字分身的完整对话记录
- 数字分身消息与用户消息左右气泡区分显示
- 初始状态展示数字分身的欢迎语，例如：「你好，我是令狐赵桓的数字分身，有什么想了解的？」

**用户输入框**
- 底部固定输入框，支持文字输入
- 包含发送按钮，点击或按回车键发送消息

**数字分身回复逻辑**

数字分身基于以下背景知识进行回复：
- 职业/身份：计算机专业的研0学生
- 正在做的事：毕设和 vibe coding
- 擅长领域：多智能体的任务规划、因果推理、大模型

预置三个常见问题及回复逻辑：

| 触发关键词/问题 | 数字分身回复逻辑 |
|----------------|------------------|
| 你叫什么名字 / 姓名 | 介绍姓名「令狐赵桓」，并说明姓名由四个字构成，令狐为复姓 |
| 网络昵称 / 你的ID | 回复网络昵称相关信息（内容由页面主人后续填充，当前以占位文案展示） |
| 内部玩笑 / 梗 | 回复预设的内部玩笑内容（内容由页面主人后续填充，当前以占位文案展示） |

- 对于未匹配预置问题的输入，数字分身返回默认回复，例如：「这个问题有点难倒我了，你可以直接联系令狐赵桓本人哦～」
- 本期数字分身回复逻辑为前端静态规则匹配，不接入外部 AI 接口

---

## 4. 业务规则与逻辑

- 页面为单页应用，所有模块在同一页面内垂直排列，无页面跳转
- 导航栏锚点点击后平滑滚动至对应模块
- 聊天区对话历史在当前会话内保留，刷新页面后重置
- 数字分身回复采用关键词匹配优先级：精确匹配 > 模糊匹配 > 默认回复
- 输入框为空时，发送按钮不可点击或点击无效

---

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户输入为空 | 禁止发送，发送按钮置灰或不响应 |
| 用户输入超长文本 | 输入框限制最大字符数（建议 200 字），超出截断并提示 |
| 未匹配任何预置问题 | 返回默认兜底回复 |
| 头像图片未配置 | 展示姓名首字母「L」作为占位头像 |
| 移动端小屏适配 | 聊天区高度自适应，输入框固定在聊天模块底部，不遮挡内容 |

---

## 6. 视觉设计规范

- **整体基调**：明亮、清新，融入适度科技感元素
- **主色调**：樱花粉（建议色值参考 #FFB7C5 或 #F9A8D4）作为主色或强调色，用于按钮、标签、分割线、气泡等元素
- **设计原则**：
  - 信息层级分明，姓名与聊天功能作为视觉重点
  - 模块间保留充足留白，避免拥挤
  - 所有文字保证对比度，确保清晰易读
- **移动端适配**：页面布局完全响应式，优先适配手机屏幕（375px 基准宽度），所有交互元素触控区域不小于 44px

---

## 7. 验收标准

- 页面可在移动端浏览器正常打开并完整展示所有模块
- 姓名「令狐赵桓」与一句话介绍在页头区域清晰可见
- 「关于我」板块四个字段（当前身份、当前主要任务、兴趣、其他）均正确展示
- 聊天区初始展示欢迎语，用户可输入文字并发送
- 三个预置问题触发后，数字分身返回对应回复内容
- 未匹配问题时，数字分身返回默认兜底回复
- 输入框为空时无法发送消息
- 页面在 375px 宽度下无横向滚动条，布局无错位
- 导航锚点可正常跳转至对应模块

---

## 8. 本期不实现功能

- 数字分身不接入外部 AI 大模型接口，仅使用静态规则匹配
- 不实现后端数据存储，聊天记录不持久化
- 不实现用户登录或身份验证
- 不实现多语言切换
- 不实现深色模式
- 不实现访客统计或埋点分析
- 网络昵称与内部玩笑的具体内容为占位文案，待页面主人后续填充
